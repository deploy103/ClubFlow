학생/멘토용 과제 관리 웹플랫폼을 기획하고 싶다.

핵심 목표:
- 학생은 로그인 후 자신이 속한 동아리와 분야 기준으로 배정된 과제만 본다.
- 멘토는 자신이 담당하는 동아리/분야에 속한 학생에게만 과제를 배정할 수 있다.
- 전체 사용자에게 글로벌하게 보이는 구조가 아니라, 소속(동아리)과 분야 기반으로 접근 및 과제 배정 범위가 제한되어야 한다.

기본 개념:
- 동아리(Club)가 있다.
- 각 동아리 안에 여러 분야(Field)가 있다.
- 학생은 특정 동아리와 분야에 속한다.
- 멘토도 담당 동아리/분야 범위를 가진다.
- 과제는 특정 동아리/분야 전체 또는 특정 학생들에게만 배정 가능하다.

필수 역할:
1. Student
2. Mentor
3. Admin

Student 기능:
- 로그인 후 자신의 소속 동아리/분야 정보 확인
- 자신에게 배정된 진행중 / 예정 / 만료 과제 확인
- 과제 상세 확인
- 제출하기
- 피드백 확인
- 질문 작성 및 답변 확인

Mentor 기능:
- 자신이 담당하는 동아리/분야 범위 내 학생 목록 조회
- 해당 범위 학생들에게 과제 생성 및 배정
- 제출물 확인
- 피드백 작성
- 질문 답변
- 미제출 학생 확인

Admin 기능:
- 동아리 생성/수정
- 분야 생성/수정
- 학생/멘토 계정 관리
- 학생 소속 배정
- 멘토 담당 범위 배정

데이터 모델에 포함할 것:
- Club
- Field
- User
- MentorScope
- Assignment
- AssignmentTarget
- Submission
- Feedback
- Question

과제 상태 기준:
- 예정됨: 시작일 이전
- 진행중: 시작일 이후, 마감일 이전
- 만료됨: 마감일 이후
- 제출완료: 학생 제출 있음
- 미제출: 제출 없음

과제 배정 방식:
- 특정 동아리 + 분야 전체 대상
- 특정 학생 개별 대상

학생 대시보드에서 보여줄 것:
- 내 동아리 / 내 분야
- 진행중 과제 수
- 예정 과제 수
- 만료 과제 수
- 새 피드백 수
- 마감 임박 과제
- 최근 피드백
- 질문 바로가기

멘토 대시보드에서 보여줄 것:
- 담당 동아리 / 분야
- 오늘 마감 과제
- 미제출 학생 수
- 최근 질문
- 피드백 작성이 필요한 제출물

디자인 요구사항:
- 세련되고 눈에 확 들어오는 UI
- 학생이 한눈에 내 과제와 마감일을 파악할 수 있어야 함
- 카드형 대시보드 중심
- 상태 배지와 마감 임박 강조
- 다크모드 기반의 프리미엄 느낌
- 포인트 컬러가 확실한 현대적 디자인
- 반응형 웹
- 깔끔하고 시각적 계층이 명확한 레이아웃
